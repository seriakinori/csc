.NET (dotnet) CLIを使用してプロジェクトを作成するための基本コマンドは **`dotnet new`** です。

以下に、よく使われる構文と具体的な使用例をまとめました。

---

### 1. 基本的な構文

```bash
dotnet new <テンプレート名> -n <プロジェクト名>

```

* **`<テンプレート名>`**: 作成したいアプリの種類（console, webapi, mvc など）
* **`-n <プロジェクト名>`**: 作成するプロジェクトの名前（フォルダも作成されます）

---

### 2. よく使う作成コマンド例

目的のアプリケーションに合わせて、以下のコマンドを実行してください。

#### コンソールアプリ（CUI）

最も基本的なアプリケーションです。

```bash
dotnet new console -n MyConsoleApp

```

#### Web API

バックエンドAPIを作成する場合に使用します。

```bash
dotnet new webapi -n MyWebApi

```

#### ASP.NET Core MVC

Webサイト（Model-View-Controller）を作成する場合に使用します。

```bash
dotnet new mvc -n MyWebsite

```

#### クラスライブラリ

他のプロジェクトから参照されるライブラリを作成する場合に使用します。

```bash
dotnet new classlib -n MyLibrary

```

---

### 3. 知っておくと便利なオプション

コマンドに追加できる主要なオプションです。

| オプション | 説明 | 例 |
| --- | --- | --- |
| **`-n`** または **`--name`** | プロジェクト名を指定します。指定しない場合、カレントディレクトリ名が使われます。 | `-n MyProject` |
| **`-o`** または **`--output`** | 出力先ディレクトリを指定します（ディレクトリがない場合は作成されます）。 | `-o src/MyProject` |
| **`-f`** または **`--framework`** | ターゲットフレームワークを指定します（特定のバージョンを使いたい場合）。 | `-f net8.0` |

---

### 4. 利用可能なテンプレートを確認する

インストールされている全てのテンプレートと、そのショートネーム（コマンドで使う名前）を確認するには、以下のコマンドを使用します。

```bash
dotnet new list

```

---

プロジェクト作成後、そのディレクトリに移動して `dotnet run` を実行するとアプリケーションが起動します。
